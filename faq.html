FAQ

What is your advice on learning C++?

I once wrote an answer on a Reddit post about a question like this.

It’s hard for me to recommend beginner C++ books, because what I did is transition from a different high-level language (Java) to C++. The usual de facto C++ beginner book that is usually recommended is C++ Primer (not to be confused with C++ Primer Plus, which is by a different author and is considered inferior). However, I did not finish the whole book, and quickly got bored of it.

What I did is read the whole C++ FAQ and then just simply start hacking on simple pet projects like a small HTTP server using Boost.Asio. Then I attended a university course about C++11 and then started reading intermediate books like the Scott Meyers series. Keep in mind, that this approach does not work for everyone.

My recommendation is this:

  1. Start reading and coding along to A Tour of C++ by none other than the creator of C++ itself, Bjarne Stroustrup.

  2. Read the C++ FAQ

  3. Practice with your initial knowledge on a side project of yours that you like. You can find ideas here: https://github.com/karan/Projects

After that, read through Scott Meyer’s Effective C++, More Effective C++, Modern Effective C++ and Effective STL. Congratulations, now you know C++! Not. Because this language has many domains (the C subset, OOP, templates, STL, C++11’s new stuff, etc.) and nobody knows all of them, people usually specialize.

What I would suggest is learn the basics of C++ first from A Tour of C++, then read the C++ FAQ, create small programming projects and read through Scott Meyer’s Effective C++ series.

If you want to see more recommended C++ books, take a look at The Definitive C++ Book Guide and List.

If you come from a different programming language, then these are the concepts that you should get yourself familiar with first:

  • Difference between the compiler and linker
  • Pointers and references
  • Difference between stack and heap memory
  • Importance of const references as function parameters from performance and safety viewpoints
  • Constructors, destructors, copy constructor and copy assignment operators
  • Difference between member initializer list and constructor body from performance viewpoint

Final tips while using C++ for the first time:

  • Do not abuse inheritance chains. Think of the maintanability nightmare of modifying base classes and learn about the performance overhead of pointer indirection. Whenever I see people making this mistake I always cite this article about the making of Starcraft on how the development team struggled with long inheritance chains when they first started using C++. Prefer composition over inheritance and don’t have more than 1-2 base classes.
  • Best way to learn is making a small project. You learn how to structure your files in your project, see the language in action and forces you to Google the problems whenever you get into roadblocks.
  • The challenge of learning a new language is not about the syntax. It’s about learning the culture and the paradigms of the language.
